# Документация второй части проекта

## Слой View

Классы слоя View работают только с DOM-разметкой. Они находят HTML-элементы в своём контейнере, сохраняют эти элементы в полях класса, отображают переданные значения и навешивают обработчики событий.

Полями View-классов являются только HTML-элементы. Данные предметной области, например `id` товара или объект товара, в View-классах не хранятся. Эти данные находятся в моделях, а обработчики с нужными данными создаются в презентере и передаются в конструкторы компонентов.

### Page

Файл: `src/components/View/Page.ts`.

Компонент главной страницы. Хранит элементы каталога, кнопки корзины, счётчика и обёртки страницы. В конструктор получает обработчик клика по корзине.

### Modal

Файл: `src/components/View/Modal.ts`.

Компонент модального окна. Хранит кнопку закрытия и контейнер содержимого. В конструктор получает обработчик закрытия. Открывается добавлением класса `modal_active`, закрывается удалением этого класса.

### Card, CatalogCard, PreviewCard, BasketCard

Файл: `src/components/View/Card.ts`.

`Card` - общий родитель карточек. Хранит элементы названия и цены, содержит общую работу с текстом, ценой и изображением.

`CatalogCard` хранит только элементы изображения и категории. `id` товара в классе не хранится. Обработчик клика создаётся в презентере внутри `map`, замыкает данные конкретного товара и передаётся в конструктор карточки.

`PreviewCard` хранит только элементы изображения, категории, описания и кнопки. Объект товара в классе не хранится. Обработчик кнопки создаётся в презентере и передаётся в конструктор. Текст и disabled-состояние кнопки задаются универсальными методами `setButton(value: string)` и `setButtonDisabled(value: boolean)`.

`BasketCard` хранит только элементы номера товара и кнопки удаления. `id` товара в классе не хранится. Обработчик удаления создаётся в презентере и передаётся в конструктор.

### Basket

Файл: `src/components/View/Basket.ts`.

Компонент корзины. Хранит список товаров, кнопку оформления и элемент итоговой цены. В конструктор получает обработчик оформления заказа. Disabled-состояние кнопки получает от презентера через параметр `buttonDisabled`.

### Form, OrderForm, ContactsForm

Файл: `src/components/View/Form.ts`.

`Form` - общий родитель форм. Хранит кнопку отправки и элемент ошибок.

`OrderForm` хранит кнопки оплаты и поле адреса. В конструктор получает обработчик изменения поля и обработчик отправки формы.

`ContactsForm` хранит поля email и телефона. В конструктор получает обработчик изменения поля и обработчик отправки формы.

### Success

Файл: `src/components/View/Success.ts`.

Компонент успешного оформления заказа. Хранит описание списанной суммы и кнопку закрытия. В конструктор получает обработчик закрытия.

## События приложения

События моделей:

- `catalog:changed` - изменился список товаров каталога.
- `preview:changed` - изменился товар, выбранный для подробного просмотра.
- `basket:changed` - изменилось содержимое корзины.
- `buyer:changed` - изменились данные покупателя.

События действий пользователя:

- `card:select` - выбрана карточка товара.
- `product:toggle` - нажата кнопка покупки или удаления товара.
- `basket:open` - нажата кнопка открытия корзины.
- `basket:remove` - нажата кнопка удаления товара из корзины; обработчик из презентера передаёт товар из параметра `item` метода `map`.
- `order:open` - нажата кнопка оформления заказа.
- `order:next` - отправлена первая форма заказа.
- `order:submit` - отправлена форма контактов.
- `form:input` - изменено поле формы.
- `modal:close` - запрошено закрытие модального окна.

Эти события генерируются в обработчиках, которые создаются в презентере и передаются компонентам View.

## Презентер

Презентер реализован в `src/main.ts`. Он создаёт модели, компоненты View, API-класс и брокер событий.

Важная часть работы презентера - создание обработчиков для View-компонентов. Например, при создании карточек каталога презентер проходит по массиву товаров из `ProductCatalogModel`, для каждого товара создаёт обработчик клика, замыкающий `item.id`, и передаёт этот обработчик в `CatalogCard`.

Так View-компонент не хранит `id`, но интерактивный элемент всё равно вызывает корректное действие для конкретного товара.

Основные сценарии:

- при `catalog:changed` презентер получает товары из модели и рендерит каталог карточками `CatalogCard`;
- при клике по карточке обработчик из презентера генерирует `card:select` с товаром из параметра `item` метода `map`;
- при `card:select` презентер сохраняет полученный товар в модели как preview;
- при `preview:changed` презентер получает товар через `catalogModel.getPreview()`, рассчитывает текст и disabled-состояние кнопки через `basketModel.hasItem()`, вызывает сеттеры `PreviewCard` и открывает модальное окно;
- при клике по кнопке подробной карточки обработчик из презентера генерирует `product:toggle` без передачи товара;
- при `product:toggle` презентер получает актуальный товар через `catalogModel.getPreview()`, затем добавляет его в корзину или удаляет из неё;
- при `basket:changed` обновляется счётчик корзины и, если корзина открыта, её содержимое;
- при `form:input` данные покупателя сохраняются в `BuyerModel`;
- при `buyer:changed` активная форма получает новые значения, ошибки и состояние кнопки;
- при `order:submit` презентер собирает `IOrderRequest`, отправляет заказ через `WebLarekApi`, очищает корзину и данные покупателя, затем показывает `Success`.
