# Документация второй части проекта

## Слой представления

Слой View отвечает только за отображение DOM и за уведомление приложения о действиях пользователя через брокер событий. Компоненты не хранят данные предметной области и не принимают решений о бизнес-логике.

### Page

Файл: `src/components/View/Page.ts`.

Компонент главной страницы. Управляет контейнером каталога, счётчиком корзины и кнопкой открытия корзины. При клике на кнопку корзины генерирует событие `basket:open`.

### Modal

Файл: `src/components/View/Modal.ts`.

Компонент модального окна. Открывает и закрывает контейнер с модификатором `modal_active`, подставляет переданную разметку в `.modal__content`. Генерирует `modal:close` по клику на крестик и по клику вне содержимого модального окна. От модального окна не наследуются другие компоненты.

### Card, CatalogCard, PreviewCard, BasketCard

Файл: `src/components/View/Card.ts`.

`Card` - общий родитель для карточек. В нём вынесена работа с названием, ценой и изображением.

`CatalogCard` отображает карточку товара в каталоге и генерирует `card:select`.

`PreviewCard` отображает подробную карточку товара в модальном окне. Кнопка получает состояние извне: `Купить`, `Удалить из корзины` или `Недоступно`. При клике генерирует `product:toggle`.

`BasketCard` отображает товар в корзине, его номер, цену и кнопку удаления. При клике на удаление генерирует `basket:remove`.

### Basket

Файл: `src/components/View/Basket.ts`.

Компонент корзины. Отображает список товаров, общую стоимость и состояние кнопки оформления. Если список пустой, используется пустой список, а текст "Корзина пуста" показывается стилями шаблона. При клике на кнопку оформления генерирует `order:open`.

### Form, OrderForm, ContactsForm

Файл: `src/components/View/Form.ts`.

`Form` - общий родитель форм. Управляет кнопкой отправки и областью ошибок.

`OrderForm` отображает первый шаг оформления: выбор оплаты и адрес. При изменении полей генерирует `form:input`, при отправке формы - `order:next`. Для выбранного способа оплаты используется модификатор `button_alt-active`.

`ContactsForm` отображает второй шаг оформления: email и телефон. При изменении полей генерирует `form:input`, при отправке формы - `order:submit`.

### Success

Файл: `src/components/View/Success.ts`.

Компонент успешного оформления заказа. Отображает списанную сумму и генерирует `modal:close` при клике на кнопку закрытия.

## События приложения

События моделей:

- `catalog:changed` - изменился список товаров каталога.
- `preview:changed` - изменился товар, выбранный для подробного просмотра.
- `basket:changed` - изменилось содержимое корзины.
- `buyer:changed` - изменились данные покупателя.

События представления:

- `card:select` - пользователь выбрал карточку товара для просмотра.
- `product:toggle` - пользователь нажал кнопку покупки или удаления товара в подробной карточке.
- `basket:open` - пользователь открыл корзину.
- `basket:remove` - пользователь удалил товар из корзины.
- `order:open` - пользователь начал оформление заказа.
- `order:next` - пользователь перешёл ко второму шагу оформления.
- `order:submit` - пользователь отправил заказ.
- `form:input` - пользователь изменил поле формы.
- `modal:close` - пользователь закрыл модальное окно.

## Презентер

Презентер реализован в `src/main.ts`, отдельный класс не создавался, потому что приложение одностраничное и вся логика сценариев компактно помещается в основном модуле.

Презентер создаёт модели, компоненты представления, API-класс и брокер событий. Он подписывается на события моделей и представления, связывает слои между собой и подготавливает данные для View. В презентере не генерируются события: он только обрабатывает события и вызывает методы моделей, компонентов и API.

Основные сценарии:

- при `catalog:changed` каталог берётся из модели и рендерится карточками `CatalogCard`;
- при `card:select` товар ищется в модели каталога и сохраняется как выбранный для просмотра;
- при `preview:changed` открывается модальное окно с `PreviewCard`;
- при `product:toggle` товар добавляется в корзину или удаляется из неё, затем модальное окно закрывается;
- при `basket:changed` обновляется счётчик корзины и, если корзина открыта, её содержимое перерисовывается;
- при `order:open` открывается первый шаг оформления;
- при `form:input` данные покупателя сохраняются в `BuyerModel`;
- при `buyer:changed` активная форма получает обновлённые значения, ошибки и состояние кнопки;
- при `order:next` открывается форма контактов;
- при `order:submit` презентер собирает `IOrderRequest`, отправляет заказ через `WebLarekApi`, очищает корзину и данные покупателя, затем показывает `Success`.
